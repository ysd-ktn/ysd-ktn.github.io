[ PHASE 7-2 / MOBILE HEADER / OPTION D ]

案 D — 常時露出ナビ

モバイルではピルを廃止し、ナビ項目を常に画面上に表示する案。
タップ操作なしで全項目が見え、認知コストがゼロになる。
D-1: 横置き（上部 2 段バー） / D-2: 縦置き（左レール）

D-1 — 横置き（上部 2 段バー）

ロゴ行 + ナビ行の 2 段構成。
項目が左右に均等分割されて常時表示。

[ D-1 評価 ]
+全項目が常に見えていて認知コストゼロ
+現在地ハイライト（アクティブ表示）が自然に組み込める
+横幅を 4 等分するだけでよく実装がシンプル
−ヘッダーが 76px を常に占有するため、Hero の見える面積が減る
−WRITING の 7 文字がやや詰まる（9px でギリギリ収まるが余白は少ない）
D-2 — 縦置き（左レール）

左端に 48px の縦レールを固定。
項目名を縦書きにしてコンパクトに収める。

[ D 案共通 — 実装上の注意 ] デスクトップ (min-width: 721px) では引き続き pill が表示される。 このヘッダーは @media (max-width: 720px) でのみ有効にし、 Header.astro の既存スタイルは @media (hover: hover) and (min-width: 721px) に閉じる形で共存可能。 どちらの案もスクロール追従（position: fixed / sticky）で動作する。
